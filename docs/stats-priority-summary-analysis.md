# 프로젝트 통계 – 담당자별 리스트 상단 “중요도별 건수 및 %” 분석

## 0. 팝업·하단 패널 크기 조정 (적용 완료)

### 요구
1. **프로젝트 통계 팝업 폭**: 15% 넓게 조정.
2. **하단 이슈 목록 패널 높이**: 15% 높게 조정.

### 현재 값
- **DialogContent**: `max-w-5xl` (= 64rem = 1024px), `max-h-[90vh]`.
- **하단 패널**(selectedGroup 표시): `max-h-64` (= 16rem = 256px), `overflow-y-auto`.

### 변경 값
- **팝업 폭**: 1024 × 1.15 ≈ 1178px → `max-w-[1180px]` 사용.
- **하단 패널 높이**: 256 × 1.15 ≈ 295px → `max-h-[295px]` 사용.

### 사이드 이펙트
| 구분 | 내용 | 판단 |
|------|------|------|
| **다이얼로그** | 통계 팝업만 넓어짐. 다른 다이얼로그(이슈 상세 드로어 등)는 별도 컴포넌트. | ✅ 프로젝트 통계 다이얼로그만 영향. |
| **작은 화면** | 90vh 높이는 유지. 폭만 1180px로 상한 확대. 작은 뷰포트에서는 여전히 뷰포트에 맞춤. | ✅ 반응형 깨짐 없음. |
| **하단 패널** | 상태 탭 내부 하단 고정 영역만 높이 증가. KPI 탭에는 해당 패널 없음. | ✅ 상태 탭 하단 패널만 영향. |

---

## 1. 대상 UI

- **위치**: 프로젝트 통계 다이얼로그 → **상태** 탭 → **담당자별 현황** 테이블.
- **동작**: 각 담당자 행에서 **전체** / 완료 / 진행 중 / 대기 / 지연 / 조기완료 / 로그 있음·없음 등 클릭 시, 다이얼로그 **하단**에 슬라이드 패널이 열림.
- **패널 구조** (현재):
  1. 상단: 제목(`selectedGroup.title`) + 총 건수 뱃지(`selectedGroup.issues.length`개) + 닫기(X).
  2. 하단: `selectedGroup.issues` 를 순회하며 이슈 카드 리스트 (키, 제목, 상태, 담당자, 완료예정일).

**요구**: 이 **리스트 최상단**에 “중요도(우선순위)별 건수 및 %” 를 표시.

---

## 2. 데이터

- **이슈 필드**: `JiraIssue.fields.priority` (optional).
  - 형태: `{ name: string, iconUrl: string }` (실제 사용 시 `id` 도 있을 수 있음).
- **조회 여부**: `getIssuesForEpic` / 이슈 검색 시 `fields` 에 `priority` 포함되어 있음 → 리스트에 쓰이는 이슈에는 이미 `priority` 가 내려옴.
- **누락 가능성**: 일부 이슈는 `priority` 가 없을 수 있음 → “미지정” 등으로 묶어서 표시 필요.

---

## 3. 표시 규칙 (제안)

- **집계 기준**: `selectedGroup.issues` 를 `issue.fields.priority?.name ?? '미지정'` 으로 그룹핑.
- **표시 내용**: 우선순위별 **건수** + **전체 대비 비율(%)**.
  - 예: `Highest 3건 (15%)`, `High 5건 (25%)`, `Medium 8건 (40%)`, `미지정 4건 (20%)`.
- **정렬**: 건수 내림차순, 또는 Jira 우선순위 순(Highest → High → Medium → Low → Lowest) 후 “미지정”. (구현 시 선택)

---

## 4. 사이드 이펙트

| 구분 | 내용 | 판단 |
|------|------|------|
| **표시 위치** | `selectedGroup` 패널은 **상태** 탭 내부에만 있음. KPI 탭·다른 탭에는 동일 패널 없음. | ✅ 상태 탭 하단 패널만 변경. 다른 탭 무영향. |
| **호출 경로** | “담당자별 전체” 뿐 아니라 **전체 이슈 / 완료 / 진행 중 / 지연** 등 상단 카드 클릭 시에도 동일 `openGroup(…)` → `selectedGroup` 사용. | ✅ 동일 패널이므로, **모든 그룹** 클릭 시 리스트 상단에 중요도 요약이 붙음. (담당자별만이 아님) |
| **데이터** | `selectedGroup.issues` 는 이미 leaf 이슈만 있음(통계가 leaf 기준이므로). priority 미포함 이슈는 `priority?.name` 으로 “미지정” 처리. | ✅ 기존 건수/통계 로직과 불일치 없음. |
| **성능** | 그룹 클릭 시 해당 이슈 배열만 그룹핑·집계하면 됨. 이슈 수가 수백 단위여도 O(n) 한 번. | ✅ 부담 없음. |
| **접근성/레이아웃** | 리스트 위에 한 줄~몇 줄 요약 블록 추가 시, 스크롤 영역(`max-h-64 overflow-y-auto`) 안에 들어가면 스크롤이 늘어남. | ✅ 요약을 리스트와 같은 스크롤 컨테이너 상단에 두면 됨. |

**정리**:  
- “담당자별 전체” 뿐 아니라 **어떤 그룹을 열어도** 리스트 최상단에 중요도별 건수·%가 보이게 됨. (일관된 동작으로 두는 것이 자연스러움.)  
- 다른 탭·다른 통계 수치에는 영향 없음.

## 5. 수정 순서 및 적용 상태

1. **집계 유틸 추가** ✅ 적용  
   - `selectedGroup.issues` 를 `priority.name`(또는 미지정) 기준으로 그룹핑.  
   - 그룹별 건수와 전체 대비 % 계산.  
   - 위치: `project-stats-dialog.tsx` 내부 `useMemo` 또는 별도 작은 함수.  
   - 우선순위 표시 순서: 건수 내림차순으로 정렬해 표시.

2. **UI 블록 추가** ✅ 적용  
   - `selectedGroup` 이 있을 때, 기존 “제목 + N개” 아래·**이슈 리스트(`.map(issue => ...)`) 위**에 새 블록 삽입.  
   - 내용: “중요도별 건수 및 %” (예: 라벨 리스트 또는 작은 테이블).  
   - 스타일: "중요도" 라벨 + 칩 형태(이름 N건 (P%)), 리스트와 같은 스크롤 영역 안 최상단.

3. **엣지 처리** ✅ 적용  
   - `selectedGroup.issues.length === 0` 인 경우: 패널 자체가 안 뜨므로 생략.  
   - 모든 이슈가 `priority` 없음: “미지정 N건 (100%)” 만 표시.  
   - `priority.name` 이 인스턴스별로 다를 수 있음(영문/한글): 있는 그대로 표시.

4. **검증**  
   - 담당자별 “전체” 클릭 시 리스트 최상단에 중요도별 건수·% 노출 확인.  
   - 상단 카드(전체 이슈 / 완료 / 지연 등) 클릭 시에도 동일하게 노출되는지 확인.  
   - priority 없는 이슈만 있는 그룹에서 “미지정”만 나오는지 확인.
